# DJ CRM v2 - CRM для музыкальной школы

Система управления музыкальной школой: клиенты, занятия, платежи, задачи.

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
# Создание виртуального окружения
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate  # Windows

# Установка зависимостей
pip install django psycopg2-binary python-environ
```

### 2. Настройка окружения

Создайте файл `.env` в корне проекта:

```env
SECRET_KEY=your-secret-key-here
DEBUG=True

# База данных
DB_NAME=djcrm
DB_USER=postgres
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=5432

# Хосты
ALLOWED_HOSTS=localhost,127.0.0.1
```

### 3. База данных

```bash
# Создание базы данных
createdb djcrm

# Применение миграций
python manage.py makemigrations
python manage.py migrate

# Создание суперпользователя
python manage.py createsuperuser
```

### 4. Запуск сервера

```bash
python manage.py runserver
```

Доступно по адресу: http://127.0.0.1:8000

## 🔄 Автоматизация

### Статусы клиентов

Система автоматически обновляет статусы клиентов на основе последнего взаимодействия:

| Статус | Условие |
|--------|---------|
| `ACTIVE` | Активный (менее 14 дней без активности) |
| `SILENT` | Молчун (более 14 дней без активности) |
| `LOST` | Потеряшка (более 30 дней без активности) |
| `CANCELLED` | Расторгнул договор |
| `POTENTIAL` | Потенциальный (еще не покупал) |

### Триггеры

- **При создании взаимодействия**: обновление `last_interaction_date` и `silent_days`
- **При создании платежа**: автоматическая генерация номера
- **При создании счета**: автоматическая генерация номера
- **При сохранении занятия**: автоматический расчет времени окончания

## 🔧 Утилиты

### Декораторы для views:

```python
from core.permissions import (
    role_required,
    permission_required,
    manager_required,
    teacher_required,
    finance_access_required,
)

@role_required('OWNER', 'ADMIN')
def admin_panel(request):
    # Доступно только владельцу и администратору
    pass

@manager_required
def client_list(request):
    # Доступно менеджерам и выше
    pass

@finance_access_required
def payment_list(request):
    # Доступно только при праве на просмотр финансов
    pass
```

### Проверка прав в шаблонах:

```django
{% if user_roles|length > 0 %}
    {% for role in user_roles %}
        <span>{{ role.name }}</span>
    {% endfor %}
{% endif %}

{% if user_can_view_finance %}
    <a href="{% url 'payment_list' %}">Платежи</a>
{% endif %}
```
