## 🎯 Описание

**DJ CRM v2** — профессиональная система управления школой музыки с современным дизайном, полной автоматизацией финансовых операций и удобной системой управления клиентами.

### ✨ Ключевые возможности

- 👥 **Управление клиентами** — профили, история взаимодействий, теги
- 📚 **Курсы и зачисления** — каталог программ, запись студентов, прогресс
- 💰 **Платежи** — полная финансовая система со статистикой и фильтрами
- 📅 **Расписание** — календарь занятий с FullCalendar
- 📝 **Задачи** — планирование и контроль выполнения
- 🎨 **Современный UI** — glassmorphism, анимации, тёмная тема

## 🚀 Быстрый старт

### Требования

- Python 3.9+
- PostgreSQL 14+ (или SQLite для разработки)
- Node.js 16+ (опционально, для CSS-сборки)

### Установка

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/lynhdnb/DJ_CRM_v2.git
cd DJ_CRM_v2

# 2. Создайте виртуальное окружение
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate     # Windows

# 3. Установите зависимости
pip install -r requirements.txt

# 4. Настройте переменные окружения
cp .env.example .env
# Отредактируйте .env (SECRET_KEY, DATABASE_URL и т.д.)

# 5. Примените миграции
python manage.py makemigrations
python manage.py migrate

# 6. Создайте суперпользователя
python manage.py createsuperuser

# 7. Запустите сервер
python manage.py runserver
```

Откройте `http://127.0.0.1:8000`

## 💰 Система платежей

### Возможности

- ✅ **Создание платежей** — разовые, банковские, школьные
- ✅ **Фильтры** — статус, клиент, способ, тип, даты, поиск
- ✅ **Статистика** — общая сумма, ожидающая, средний чек
- ✅ **Счета (Invoices)** — генерация номеров `INV-{YEAR}-{NUMBER}`
- ✅ **Договоры (Contracts)** — пакеты уроков, рассрочки
- ✅ **Возвраты** — полный контроль финансовых операций

### API эндпоинты

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/payments/` | Список платежей с фильтрами |
| GET | `/payments/create/` | Форма создания |
| POST | `/payments/create/` | Создание платежа |
| GET | `/payments/<id>/` | Детали платежа |
| POST | `/payments/<id>/pay/` | Подтверждение оплаты |
| POST | `/payments/<id>/delete/` | Удаление |

## 👥 Управление клиентами

### Структура профиля

- **Профиль** — фото, имя, статус, баланс (уроки/практика)
- **Контакты** — телефон, email, дополнительные контакты
- **Платежи** — история с суммами и статусами
- **Записи** — активные курсы, прогресс
- **Лог взаимодействий** — чат с комментариями, задачами, звонками

### Статусы клиентов

| Статус | Описание |
|--------|----------|
| NEW | Новый клиент |
| ACTIVE | Активный студент |
| SILENT | Неактивный (нет занятий > 30 дней) |
| LOST | Потерянный клиент |
